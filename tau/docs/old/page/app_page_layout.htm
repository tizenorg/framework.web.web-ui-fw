Application Page

Page Layout

In the Samsung Gear Web UI framework, the application page structure is based on header, content elements

The header is placed at the top, and displays the page title. The content is the area below the header, showing the main content of the page.

To add a page to the application, use the following code:

<!--Page Layout-->

<div class="ui-page ui-page-active">
 <header class="ui-header"></header>
 <div class="ui-content"></div>
 <footer class="ui-footer"></footer>
</div>

<!--Page Layout with more button in header-->
<div class="ui-page ui-page-active">
  <header class="ui-header ui-has-more">
   <h2 class="ui-title">Call menu</h2>
   <button type="button" class="ui-more ui-icon-overflow">More Options</button>
  </header>
  <div class="ui-content">
	Content message
  </div>
  <footer class="ui-footer">
   <button type="button" class="ui-btn">Footer Button</button>
  </footer>
</div>

Multi-page Layout

This template contains multiple "page" containers inside, unlike a single page template that has just one page within it.

the main page is set up the page that has "ui-page-active" class. If there is no one that has "ui-page-active" class, the page what is first in source order set up main page. but it's good to set up "ui-page-active" class at least one page for performance.

You link to internal pages by referring to the id of the page you want to show. For example, to link to the page with an id of "two", my link would have a href="#two" in the code. 

<!--Main Page-->
<div id="one" class="ui-page ui-page-active">
 <header class="ui-header"></header>
 <div class="ui-content"></div>
 <footer class="ui-footer"></footer>
</div>

<!--Secondary Page-->
<div id="two" class="ui-page">
 <header class="ui-header"></header>
 <div class="ui-content"></div>
 <footer class="ui-footer"></footer>
</div>

To find the current active page, use the 'ui-page-active' css class.

Page Interface

Interface Description
gear.ui.changePage ( to ) Programmatically change to another page. argument 'to' is a page object or string.
gear.ui.back() This method loads the previous page in the history list.

Page Events

Event name Description
pagebeforecreate Triggered on the page being initialized, before most plugin auto-initialization occurs.
pagecreate Triggered when the page has been created in the DOM (via ajax or other) but before all widgets have had an opportunity to enhance the contained markup.
pagebeforeshow Triggered on the “toPage” we are transitioning to, before the actual transition animation is kicked off.
pageshow Triggered on the “toPage” after the transition animation has completed.
pagebeforehide Triggered on the “fromPage” we are transitioning away from, before the actual transition animation is kicked off.
pagehide Triggered on the “fromPage” after the transition animation has completed.
pagechange This event is triggered after the changePage() request has finished loading the page into the DOM and all page transition animations have completed.

Binding an event callback on the Back key

/* JavaScript code */
window.addEventListener( 'tizenhwkey', function( ev ) {
	if( ev.keyName == "back" ) {
		/* Call window.history.back() to go to previous browser window */
		/* Call window.close() to exit application */
		/* or add script to add another behavior */
	}
}

Page Transitions

By default, the framework doesn't apply transition. To set a custom transition effect, add the data-transition attribute to the link. 

<a href="index.html" data-transition="slideup">I'll slide up</a>

Global configuration 

/* JavaScript code */
gear.ui.defaults.pageTransition = "slideup";
option Description
none default
slideup
